Most of my work has been backend: Django services, REST, GraphQL APIs, SOAP, databases. That's where I've spent the most time. On databases, I've worked with PostgreSQL across all three companies and SQL Server at Inoa, where we also ran a large SQL Server to PostgreSQL migration. Beyond that, I've worked with async workers (Celery), Redis, RabbitMQ at Buser and Inoa, event-based systems like fanout messaging and PGpubsub, queue-based sync communication at Inoa, and file processing pipelines at Eden.

On frontend: most of it was at Buser, Vue 2 and Vue 3 on a PWA, helped migrate part of it to Flutter, and did some React. At Inoa I built some Django admin screens. I also touched some React at Eden.

On infrastructure, I've done a fair amount: CI/CD pipelines (migrated Jenkins to GitHub Actions at Inoa, GitLab CI at Buser), some Terraform at both Inoa and Eden, Docker, Kubernetes, and AWS across all three companies. I also managed self-hosted tooling like Sentry and Grafana at more than one company. Not a dedicated DevOps role, but I've never needed handholding there.

At Inoa my team owned platform uptime across all client-facing systems. I was the person called when something broke in a way nobody else could diagnose. At Buser as tech lead I owned incident response across four apps. At Eden the focus was more on performance. I worked on core flows that needed to stay fast under load.
